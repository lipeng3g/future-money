# future-money 项目计划

## 当前阶段定位
- 未来几天专注本地功能与产品打磨
- 暂停在线登录、用户体系、云端数据、D1、邮箱验证码、多端同步
- 目标不是简单修 bug，而是按“项目负责人”视角持续推进产品价值、功能设计与迭代优先级

## 当前工作流
- 长期工作器：tmux 会话 `future-money-dev`
- 对外汇报：每小时一次摘要
- 持续上下文文件：本文件 + `WORKLOG.md` + `NEXT_TASK.md`

## 近阶段优先级（本地优先）
1. 包体积与首屏性能
2. 关键数据模型与校验边界稳固
3. 事件/时间线/图表体验打磨
4. 导入导出与迁移兼容性
5. AI 分析体验、安全与可理解性
6. 文档、测试与可维护性提升

## 决策原则
- 优先做独立可验证、能直接提升产品质量的改动
- 每轮至少产出：代码 / 测试 / 文档 / 明确可追踪的拆分结论之一
- 若工作需要外部平台权限，则先跳过，转做本地高价值事项
