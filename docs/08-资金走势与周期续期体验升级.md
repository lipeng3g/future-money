# 08 · 资金走势与周期续期体验升级

> 基于真实使用反馈，解决「历史范围不易发现」「周期组到期后无法继续生成」「资金曲线缺少成熟财务产品的时间语义与操作反馈」三类问题。

## 一、目标与边界

### 1.1 本期目标

1. 用户进入首页即可同时看到近期历史与未来预测，并能明确区分历史、今天和预测区间。
2. 已生成完成的周期组可以在原组内继续生成，不必新建重复周期组。
3. 曲线减少常驻噪声，强化总资产、预测区间和悬停信息，交互更接近成熟 C 端财务产品。

### 1.2 本期不做

- 不更换 VChart，不引入新的图表依赖。
- 不恢复账户起始日之前不存在的历史数据；该区间展示为「无数据」，而不是伪造为 0。
- 周期续期只沿用原账户、频率、间隔、金额、分类和备注；「从下一期起修改规则」留待后续。
- 不引入后台、云同步或自动记账。

## 二、资金走势时间范围

### 2.1 默认范围

- 默认从「今后 1 年」改为「过去 3 个月～今后 1 年」。
- 旧版仍停留在默认「今后 1 年」的本地设置执行一次性迁移；用户主动选择的其他范围不受影响。
- 保留过去/未来长周期和自定义日期范围。
- 工具栏提供「前后一年」快捷入口，历史范围无需先猜测下拉框内容。
- 时间范围名称必须显式包含「过去」和「今后」，避免用户误以为曲线只能查看未来。

### 2.2 数据语义

- 账户起始日前没有余额事实，曲线不输出该账户的数据点。
- 总资产仅汇总当天已经生效的账户；不会因为某账户尚未建立而制造一段 0 余额线。
- 「今天」是历史与预测的分界：今天及以前属于已发生，今天以后属于预测。

### 2.3 视觉与交互

- 图表绘图区显示「今天」分界线和标签。
- 未来区间使用浅色背景提示「预测」。
- 数据点默认隐藏，仅在悬停时强调，避免日粒度下满屏圆点。
- 总资产线比账户线更醒目；账户线保留品牌色但降低视觉权重。
- Tooltip 显示日期、各曲线余额，并明确标记「历史」或「预测」。
- 保留底部范围滑块、拖拽平移和「重置缩放」；鼠标滚轮继续用于页面滚动，不劫持滚轮缩放。

## 三、周期组继续生成

### 3.1 入口

在「管理周期组」弹窗底部增加主操作「继续生成」。弹窗同时展示：

- 当前组生成笔数；
- 已生成至哪个日期；
- 继续生成的期数；
- 新增日期范围预览。

### 3.2 生成前数据预览

- 首次创建周期组和继续生成都必须展示即将写入的真实记录列表，再允许确认。
- 每行至少显示序号、日期和带正负号的金额；账户、分类、备注沿用当前表单/周期组上下文，不重复堆叠。
- 常见的 12～60 期完整展示；超过 100 期时展示前 80 期和后 20 期，并明确提示中间省略数量及实际总生成笔数。
- 首次创建的主按钮显示「生成 N 笔」；继续生成使用独立确认弹窗，避免误触后直接写入。
- 空预览或达到安全上限时禁止静默生成，必须给出明确提示。

### 3.3 频率表达

- 废弃容易歧义的「频率：每月；间隔：1」组合文案。
- 改为「周期单位：按月；重复规则：每 1 个月一次」。
- 同时展示前三个实际日期，例如：`每 1 个月一次：1 月、2 月、3 月`；当数值为 2 时显示：`每 2 个月一次：1 月、3 月、5 月`。
- 日、周、季度、半年、年使用同一表达规则：「每 N 天/周/个季度/个半年/年一次」。
- 周期管理摘要复用相同格式，不再显示「每月（每 1 次）」这类内部参数式表达。

### 3.4 生成规则

1. 以周期元数据的已生成边界与现存最新记录中更晚者为准，从原周期序列的下一期开始追加。
2. 新记录继续使用原 `seriesId`，在账本中仍属于同一个周期组。
3. 默认追加 12 期，允许用户修改为 1～5000 期。
4. 新记录沿用周期组元数据：账户、金额、频率、间隔、分类、备注。
5. 只向最新日期之后追加，不回填用户曾主动删除的历史空洞。
6. 以 `seriesId + date` 去重，防止重复生成。
7. 成功后更新周期组结束条件，并提示「已新增 N 笔，生成至 YYYY-MM-DD」。

### 3.5 数据层接口

Store 新增：

```ts
extendRecurring(seriesId: string, count: number): {
  added: number;
  first?: string;
  last?: string;
}
```

数据层负责校验周期组存在、计算下一批日期、追加真实 `Transaction`、去重和更新 `Series.end`；组件不自行拼交易记录。

## 四、第一阶段验收标准

### 4.1 时间范围与曲线

- [x] 首次打开默认范围为「过去 3 个月～今后 1 年」。
- [x] 可以选择「过去 1 年～今后 1 年」、长周期和自定义范围。
- [x] 账户起始日前不绘制伪 0 余额线。
- [ ] 图中能看见「今天」分界和未来预测背景。
- [ ] 常态不显示密集圆点；悬停仍能定位数据。
- [ ] Tooltip 能区分历史与预测。
- [ ] 页面滚轮不被图表劫持，滑块缩放与重置仍可用。

### 4.2 周期续期

- [x] 管理周期组弹窗显示当前最新日期。
- [x] 可输入继续生成期数，默认 12。
- [x] 提交前显示新增笔数和起止日期预览。
- [x] 确认后新记录归入原周期组，不创建第二个组。
- [x] 连续点击续期不会产生重复日期。
- [x] 月末、季度、半年、年度周期继续遵循原日期对齐规则。
- [x] Store 与周期算法测试覆盖正常续期、被删历史空洞、去重和上限。
- [x] 首次生成前显示待生成记录列表，确认按钮明确显示笔数。
- [x] 继续生成先打开独立确认弹窗并显示待生成记录列表。
- [x] 「每 N 个周期一次」文案和前三个日期示例一致，不再使用有歧义的「间隔」。

## 五、后续候选

- 从下一期开始修改金额、分类或备注，并保留历史规则。
- 上一时间窗口 / 下一时间窗口 / 回到今天。
- 收支事件标记与右侧明细抽屉。
- 按跨度自动选择日/周/月粒度。
- Tooltip 增加较上一期变化和当日收支摘要。
