# 01 · 需求与产品设计

> FutureMoney —— 个人资金的「未来推演」工具。
> 通过把每一笔（含周期性）资金变动记录为**真实可编辑的记录**，直观推演未来数月、数年的资金走势。

## 1. 背景与定位

很多人想知道：按现在的收支节奏，**未来某个时间点我手里到底有多少钱**？

- 现金、理财、储蓄等多个账户分散，难以汇总；
- 工资、房贷、定投等是**周期性**发生的，心算很难；
- 一次性的大额收支（年终奖、买车、装修）会打乱节奏。

FutureMoney 解决的核心问题：**把未来的钱看清楚，并且能随时调整。**

## 2. 旧版痛点（本次推翻重做的根本原因）


| 痛点          | 说明                                  | 新版对策                          |
| ----------- | ----------------------------------- | ----------------------------- |
| 周期资金是「算出来的」 | 旧版周期性收支在渲染时临时计算，不存在真实记录             | 创建时即**展开为一笔笔真实记录**，每笔独立可改可删   |
| 单笔无法修改      | 例如 1 月创建月度定投，6 月想删掉 5 月那笔做不到（它是虚拟的） | 每笔都有独立 `id`，任意一笔（过去/未来）都能单独操作 |
| 展示与操作不便     | 信息密集、交互绕、像后台管理系统                    | 现代化前台体验：大图为中心、卡片化、浮层引导        |


## 3. 目标用户与典型场景

**目标用户**：希望规划个人/家庭现金流的普通人，非财务专业。

**典型场景**：

1. **建账户**：新建「招行现金」账户，初始余额 ¥20,000，起始日 2026-06-01。
2. **记周期收入**：每月 10 号到账工资 ¥15,000，重复 24 个月 → 系统生成 24 笔真实记录。
3. **记一次性支出**：2026-12 年终奖 +¥30,000；2027-03 装修 −¥80,000。
4. **看未来**：曲线立刻显示总资产与各账户在未来 1~3 年的走势。
5. **改某一笔**：发现 2026-09 工资会涨，点开那一笔单独改成 ¥16,000。
6. **批量删**：换工作了，把「工资」这组从 2027-01 起的记录批量删除。
7. **备份**：导出 JSON 存档，换设备时导入恢复。

## 4. 功能范围（本期）

### 4.1 必做（MVP 核心闭环）

- **账户管理**：增删改、初始金额 + 起始日期、配色、归档。
- **资金变动**：单笔录入（账户/日期/方向/金额/分类/备注）。
- **周期录入**：频率（日/周/月/季/半年/年/一次性）+ 间隔 + 结束条件（截止日期或次数），创建时展开为真实记录。
- **图表**：总资产曲线 + 各账户曲线（可显隐/切换），日/周/月粒度，时间范围可调。
- **点击某天**：在曲线上点击日期 → 查看/新增/修改/删除当天该账户的变动。
- **表格视图**：账本明细，按月分组，可筛选、行内改删。
- **批量管理**：对一个周期「系列」，按时间列出全部记录、默认全选、自由勾选后批量删/改。
- **分类/标签**：给资金变动归类（工资、房贷、投资…）。
- **备份**：JSON 导入 / 导出。
- **主题**：深色 / 浅色切换。

### 4.2 暂不做（明确排除）

- AI 分析（旧版功能，本期不做）。
- 多币种、汇率换算。
- 多语言（仅中文）。
- 云端账号 / 多设备实时同步（仅本地 + 手动备份）。

## 5. 非功能性要求

- **真实可编辑**：任何展示在曲线/表格上的数据都对应一条可操作的真实记录。
- **现代化前台体验**：见《04 界面与交互》，避免后台管理系统观感。
- **离线可用**：纯前端，数据存本地，无网可用。
- **可维护**：TypeScript 全量类型；单文件 ≤ 400 行；核心算法有单元测试。
- **性能**：常见数据量（数千~上万笔）下曲线交互流畅。

## 6. 术语表


| 术语             | 含义                              |
| -------------- | ------------------------------- |
| 账户 Account     | 一个资金容器，有初始余额与起始日                |
| 变动 Transaction | 一笔真实的资金增减记录（存入为正、取出为负）          |
| 系列 Series      | 由一次周期性录入产生的一组变动的元数据，用于批量定位/管理   |
| 分类 Category    | 给变动打的标签                         |
| 余额 Balance     | 账户在某日的累计资金 = 初始余额 + 该日及之前所有变动之和 |
| 总资产            | 所有账户在某日余额之和                     |
