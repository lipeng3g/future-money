## M0 · 脚手架
- 初始化 Vite + React + TS 工程；安装 Semi、VChart、Zustand、dayjs。
- 配置 `vite.config.ts`（`@` 别名、vitest、端口）、`tsconfig`、ESLint。
- 接入 Semi 主题与全局样式；搭好 `App.tsx` 三区骨架与 `AppHeader`。
- **验收**：开发服务器启动，显示带 Header 的空布局，可切换深浅主题。

## M1 · 数据层（算法优先 + 测试）
- 定义 `types/*`。
- 实现 `utils`：`id`、`money`、`date`、`recurrence`、`balance`、`aggregate`、`backup`。
- 实现 `store/useStore` 与各 slice、`persist`。
- 编写并通过单测（见《03》测试清单）。
- **验收**：`vitest run` 全绿；可在控制台调用 action 操作数据并持久化。

## M2 · 账户与分类
- `AccountPanel` + `AccountCard` + `AccountFormModal`：增删改、初始余额/日期/配色、归档。
- `CategoryManager` + `CategoryFormModal`。
- `EmptyState` 引导（无账户首屏）。
- **验收**：能创建多个账户，卡片显示当前余额；能管理分类。

## M3 · 图表与表格
- `useChartData`：由 store 派生总资产线 + 各账户线，按粒度聚合。
- `BalanceChart`（VChart 折线，多系列）+ `ChartToolbar`（时间范围/粒度/图例显隐）+ `OverviewStats`。
- `LedgerTable`：按月分组、筛选、行内操作占位。
- **验收**：账户初始余额体现在曲线与表格；切换粒度/范围/显隐生效。

## M4 · 变动录入
- `TransactionFormModal`：单笔 + 周期（频率/间隔/结束条件 + 生成预览）。
- 周期通过 `expandRecurrence` 展开为真实 `Transaction` + `Series`。
- `DayDetailModal`：点击曲线某天 → 当日明细 → 增删改。
- 表格行内「改/删」接通。
- **验收**：录入月度工资生成 N 笔真实记录；曲线即时变化；可单独改/删任意一笔（含未来）。

## M5 · 批量管理
- `SeriesManageModal`：按时间列出整组、默认全选、快捷（全选/反选/仅未来/仅过去）、自由勾选。
- `batchDeleteTransactions` / `batchUpdateTransactions` 接通。
- 二次确认与结果反馈。
- **验收**：可对一组周期记录批量删除或批量改金额/分类；曲线表格即时刷新。

## M6 · 备份与打磨
- `backup`：导出 JSON、导入校验/迁移/覆盖或合并、预览摘要。
- 主题切换联动 VChart；空状态/加载/错误反馈完善。
- 性能：大数据量下 `useMemo` 缓存、必要时虚拟滚动表格。
- 文档与 README 同步。
- **验收**：完整使用闭环；导出再导入数据一致；体验符合现代化前台基调。

## 进度跟踪约定
- 每完成一个里程碑，更新本文件对应阶段为「已完成」，并在提交信息中标注（如 `feat: M3 图表与表格`）。
- 偏离设计或新增需求时，先更新对应 docs，再实现。

## 风险与对策
| 风险 | 对策 |
| --- | --- |
| 周期展开数据量过大 | `MAX_OCCURRENCES` 上限 + 录入预览提示 |
| localStorage 容量瓶颈 | 抽象 storage 适配层，必要时切 IndexedDB |
| VChart 与 Semi 主题联动 | 统一 `useTheme` 驱动二者色板 |
| 单文件膨胀 | 严格 ≤400 行，超出即拆分 |
