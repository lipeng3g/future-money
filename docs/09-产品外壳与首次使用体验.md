# FutureMoney 产品外壳与首次使用体验升级

> 本文定义第三阶段的页面产品化改版。目标不是增加后台式菜单，而是在不改变资金数据模型与核心业务流程的前提下，让页面结构、长任务承载方式和产品信息更像一套稳定、可信、可长期使用的正式产品。

## 一、现状问题

1. 桌面端账户侧栏固定占用 280px，用户查看长周期图表或账本时无法主动释放空间。
2. 顶栏把分类、导出、导入、示例数据和清空数据全部平铺，高频、低频与危险操作没有层级。
3. Logo、产品名与副标题在同一行横向堆叠，品牌区与操作区缺少清晰分组。
4. 记账、周期创建、周期组管理等长任务仍使用居中弹窗，内容一多就形成“弹窗套滚动区”。
5. 周期预览虽然限制了最大高度，但默认仍占据较多纵向空间，用户难以快速确认规则与总笔数。
6. 首次进入只有空状态或示例数据，没有稳定的使用说明入口。
7. 页面没有版本、数据存储说明、Copyright 与 GitHub 等基本产品信息。

## 二、设计原则

- **图表优先**：桌面端允许收起账户侧栏，把横向空间还给走势图与账本。
- **操作分级**：高频操作直接可见；导入、导出、示例和清空收进“数据与设置”。
- **长任务用抽屉**：需要填写、预览或批量处理的任务使用右侧工作抽屉；短编辑与二次确认继续使用 Modal。
- **底部操作稳定**：抽屉使用固定底部操作区，正文独立滚动，按钮与窗口底部保持规范间距。
- **先摘要后细节**：周期预览默认展示紧凑窗口、总笔数与起止日期，需要时再展开查看完整列表。
- **正式但不过度**：增加帮助、关于与产品信息，不引入后台导航、用户中心或无实际功能的设置页。

## 三、应用骨架

### 3.1 可收缩账户侧栏

- 仅桌面宽度（大于 1024px）支持手动展开/收起。
- 展开宽度保持 280px；收起后宽度为 64px。
- 品牌标识与产品名归入侧栏顶部；收缩控制器放在侧栏底部，避免干扰品牌区和主顶栏。
- 展开时控制器显示“收起账户栏”轻量操作行；收缩后只保留方向图标与悬停提示。
- 展开态账户由独立卡片改为轻量列表行，减少边框和阴影；每行保留账户首字图标、名称与余额。
- 收起后保留带账户首字的彩色图标，悬停显示账户名称和当前余额；点击仍可聚焦账户。
- “全部账户”和“新建账户”保留图标入口。
- 收缩状态写入本机偏好，下次打开继续沿用。
- 1024px 及以下继续使用横向账户条，不继承桌面收缩样式。

### 3.2 双区顶栏信息架构

- 左侧品牌区与账户栏严格同宽；展开时显示 Logo、产品名和定位，收起时只显示 Logo。
- 主内容顶栏不使用固定的“资金总览”，而是显示当前查看范围：全部账户、具体账户或尚未创建账户。
- 全部账户时显示账户数量；聚焦账户时复用账户首字与颜色，并直接提供“查看全部”入口。
- 右侧将“记一笔 / 新建账户”作为唯一主操作，保留分类管理、帮助、GitHub、主题切换和“数据与设置”。
- 导入、导出、载入示例数据、清空全部数据统一收进“数据与设置”菜单。
- 清空数据不再以常驻红色按钮占据顶栏，但仍保留危险色与二次确认。
- 移动端只保留必要图标和一个收纳菜单，品牌副标题自动隐藏。
- 移动端恢复单一顶栏，账户列表位于其下方横向滚动，不显示桌面收缩控制器。

### 3.3 产品信息

侧栏底部和使用说明中提供：

- `© 2026 FutureMoney`
- `v1.0.0`
- “数据仅保存在本机”
- GitHub：<https://github.com/lipeng3g/future-money>

## 四、弹层规范

### 4.1 使用右侧抽屉

- 记一笔 / 编辑变动 / 创建周期记录。
- 管理周期组及其批量操作。
- 分类管理。
- 首次使用说明与帮助。

抽屉正文独立滚动，页脚固定；桌面端使用 480～720px 合理宽度，移动端不超过视口宽度。

### 4.2 保留居中 Modal

- 新建或编辑账户。
- 新建或编辑单个分类。
- 导入数据确认。
- 删除、清空、覆盖等短二次确认。

### 4.3 周期续期流程

- 周期组管理与“确认继续生成”在同一个抽屉中切换步骤，避免抽屉上再叠一个长 Modal。
- 确认步骤提供明确的“返回修改”和“生成 N 笔”。
- 新批次金额、方向、分类、备注继续沿用第二阶段能力。

## 五、周期预览压缩

- 预览头部始终显示总笔数、首期和末期。
- 默认预览窗口控制在约 120px，只露出少量记录，但真实记录仍可滚动检查。
- 提供“展开预览 / 收起预览”按钮；展开后最大高度约 300px，不推动整个抽屉无限增长。
- 超过 100 笔仍采用前 80、后 20 的安全展示策略，并明确省略数量。
- 生成按钮始终显示真实总数，预览收起不影响校验与生成结果。

## 六、首次使用说明

首次进入自动展示一次使用说明，并可以从顶栏帮助入口随时重新打开。内容控制在四步：

1. 创建或检查账户与期初余额。
2. 记录单笔收支，或先预览再生成周期记录。
3. 通过历史/未来时间范围查看资金走势，点击曲线查看当日明细。
4. 定期导出 JSON 备份；所有数据仅保存在当前浏览器本机。

关闭首次说明后记录本机偏好，不在后续启动时反复打扰。

## 七、本阶段不做

- 不增加登录、云同步、团队协作或服务端账户体系。
- 不新增多级后台导航、独立设置路由或复杂新手任务系统。
- 不改变交易、周期组、图表范围与导入导出的业务数据结构。
- 不把所有短确认强制改成抽屉。

## 八、验收标准

- [x] 桌面侧栏可手动收起/展开，刷新后保持状态；移动端布局不受影响。
- [x] Logo、侧栏控制器和账户列表构成统一侧栏区域，主顶栏只承载页面上下文与全局操作。
- [x] 展开态使用轻量账户列表，收缩态使用账户首字图标，账户聚焦反馈在主顶栏同步显示。
- [x] 顶栏不再平铺导入、导出、示例和清空，危险操作保持二次确认。
- [x] 品牌区、主操作和辅助入口分组清晰，在窄屏下不溢出。
- [x] 记账、周期组管理和分类管理使用右侧抽屉，底部按钮有稳定留白。
- [x] 周期预览默认紧凑，可展开检查完整内容，生成数量始终准确。
- [x] 周期续期在同一抽屉中完成“配置 → 预览确认 → 生成”。
- [x] 首次进入自动展示一次说明，帮助入口可以再次打开。
- [x] 页面可找到本地存储说明、Copyright、版本号和 GitHub 地址。
- [x] 第二阶段的续期参数修改、Store 测试、类型检查和生产构建保持通过。
