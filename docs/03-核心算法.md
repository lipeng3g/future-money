# 03 · 核心算法

> 三大核心：**周期展开（recurrence）**、**余额计算（balance）**、**曲线聚合（aggregate）**。
> 全部为纯函数，输入确定则输出确定，便于单元测试。金额单位为「分」（整数）。

## 1. 周期展开 recurrence

将一次「周期录入」展开为一组真实 `Transaction`。

### 1.1 输入 / 输出

```ts
interface RecurrenceInput {
  accountId: string;
  frequency: Frequency;
  interval: number;          // ≥ 1
  baseAmount: Money;         // 有符号
  startDate: string;         // YYYY-MM-DD
  end: RecurrenceEnd;        // count 或 until
  categoryId?: string;
  note?: string;
}

// 输出：一个 Series（元数据） + N 条 Transaction（真实记录）
function expandRecurrence(input: RecurrenceInput):
  { series: Series; transactions: Transaction[] };
```

### 1.2 频率 → 日期步进

以 `startDate` 为基准，第 `k`（从 0 开始）笔的日期：


| frequency  | 第 k 笔日期                    |
| ---------- | -------------------------- |
| once       | 仅 k=0，即 startDate          |
| daily      | startDate + k·interval 天   |
| weekly     | startDate + k·interval 周   |
| monthly    | startDate + k·interval 月   |
| quarterly  | startDate + k·3·interval 月 |
| semiannual | startDate + k·6·interval 月 |
| annual     | startDate + k·interval 年   |


> 使用 `dayjs(startDate).add(...)` 计算。

### 1.3 月末对齐规则（重要边界）

按「月/季/半年/年」步进时，若起始日为 31 号而目标月份无 31 号：

- 采用 **clamp 到当月最后一天**（dayjs `add(months)` 默认行为）。
- 例：1/31 → 2/28（或 2/29）→ 3/31 …
- 在 04 文档与 UI 中提示用户该规则，避免困惑。

### 1.4 终止条件

- `count`：生成 `count` 笔（k = 0 .. count-1）。
- `until`：当第 k 笔日期 ≤ `until` 时继续，超过即停止。
- 安全上限：单次展开最多 `MAX_OCCURRENCES`（如 5000），超出给出告警，防止误操作（如日频 + 几十年）。

### 1.5 伪代码

```ts
function expandRecurrence(input) {
  const series = makeSeries(input);
  const txs: Transaction[] = [];
  for (let k = 0; k < MAX_OCCURRENCES; k++) {
    const date = stepDate(input.startDate, input.frequency, input.interval, k);
    if (input.end.kind === 'count' && k >= input.end.count) break;
    if (input.end.kind === 'until' && date.isAfter(input.end.date)) break;
    txs.push(makeTransaction({
      accountId: input.accountId,
      date: date.format('YYYY-MM-DD'),
      amount: input.baseAmount,
      categoryId: input.categoryId,
      note: input.note,
      seriesId: series.id,
    }));
    if (input.frequency === 'once') break;
  }
  return { series, transactions: txs };
}
```

## 2. 余额计算 balance

### 2.1 单账户某日余额

```
balance(account, D) = account.openingBalance
                    + Σ amount  (该账户所有 date ≤ D 的 transaction)
仅当 D ≥ account.openingDate，否则视为账户尚未存在（返回 null / 不绘制）
```

### 2.2 高效构建「每日余额序列」

对单账户在 `[from, to]` 区间生成逐日余额：

1. 过滤该账户的 transactions，按 `date` 升序。
2. 设 `running = openingBalance`，先累加所有 `date < from` 的变动，得到区间起点余额。
3. 从 `from` 到 `to` 逐日推进：把当日的变动累加进 `running`，记录当日 `running`。
4. 复杂度 O(T + Days)，T 为该账户变动数。

```ts
function dailyBalances(account, txs, from, to): { date: string; value: Money }[];
```

### 2.3 总资产序列

对所有未归档账户分别求逐日余额后**按日相加**。账户在其 `openingDate` 之前不贡献余额（按 0 计或不参与，统一约定为 0）。

```ts
function totalDailyBalances(accounts, txs, from, to): { date: string; value: Money }[];
```

## 3. 曲线聚合 aggregate

曲线按粒度展示，避免日频数据点过多。

### 3.1 粒度

- `day`：每天一个点（余额取当日值）。
- `week`：每周一个点，取**周末当天余额**（周期末快照）。
- `month`：每月一个点，取**月末当天余额**。

> 余额是「时点快照」，聚合取**区间末值**而非求和/平均，才符合「那一刻有多少钱」。

```ts
function aggregate(daily: {date; value}[], granularity): { label: string; value: Money }[];
```

### 3.2 时间范围

- 由设置 `rangePreset` 解析为 `[from, to]`，如「过去 1 月 ~ 未来 12 月」。
- 默认：`from = 今天 - 1 月`，`to = 今天 + 12 月`。

## 4. 表格数据派生

账本表格直接来自 `transactions`：

- 按 `date` 排序，按「年-月」分组；
- 每行展示：日期、账户、分类、金额（带正负色）、备注、所属系列标记；
- 支持按账户 / 分类 / 时间范围筛选。

## 5. 批量操作的数据逻辑

见《04 界面与交互》的交互；数据侧：

- **批量删除**：对选中的 `transactionId[]` 执行删除。
- **批量修改**：对选中的 `transactionId[]` 应用补丁（如金额、分类、备注）。
- 二者均为对 `transactions` 数组的纯增删改，立即触发余额/曲线重算。

## 6. 单元测试清单（Vitest）

- `recurrence`：各频率步进、interval、count/until 终止、月末 clamp、once、超上限告警。
- `balance`：openingDate 边界、负余额、区间起点累加、空交易。
- `aggregate`：day/week/month 末值取值、跨月边界、空数据。
- `money`：元↔分转换、格式化、四舍五入。
- `backup`：导出/导入往返一致、版本迁移。
