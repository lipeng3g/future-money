# Worker follow-up · 2026-03-09 · 事件表单交互语义

本轮自治 worker 未继续强拆 `chart-base`，原因是构建产物确认其主要是 ECharts 共享运行时本体，而不是简单的本地工具函数拼接；继续硬拆收益不确定，且可能引入图表能力回退。

因此本轮转做同样高价值、但更稳妥可闭环的本地任务：

- 把事件表单“频率切换 → 展示哪些字段 / 自动补哪些默认值”的交互语义下沉到 `src/utils/event-form.ts`
- 为这些交互规则补 Vitest 回归测试，减少后续组件重构时的行为漂移

本轮新增的稳定性收益：

- 组件模板不再直接散落复杂的频率判断
- `once / monthly / quarterly / semi-annual / yearly` 的字段显隐规则有了单一真源
- 频率切换时的默认值补齐逻辑有了可测试的纯函数入口
- 后续如果再补组件级 UI 测试，可直接复用这层语义约束
