# 07 · 交互与版面优化修复

> 针对实际使用中暴露的三类体验问题进行根因修复，并记录方案与验证结果。

## 一、问题与根因

| # | 用户反馈 | 根因定位 |
| --- | --- | --- |
| 1 | 走势图不能直观滚动/缩放，放大看某天、点击改删做不到，范围不灵活 | `BalanceChart` 的 `dataZoom.roamZoom=true` 让鼠标滚轮被图表劫持：页面无法下滑，且滚轮缩放越界后曲线整体消失（空白） |
| 2 | 账户面板占大量版面，导致数据展示不全 | 概览卡片（`padding 16px`、数值 `24px`）+ 账户卡片换行（`flex-wrap: wrap`）+ 图表卡片固定 `clamp(440px,64vh,860px)`，账本明细被挤到首屏外，又因问题 1 无法滚动 |
| 3 | 缺少预置分类，分类弹窗底部按钮贴边难看 | `CategoryManager` 仅在「首次进入」自动 seed；`清空` 后无分类入口；footer 用整宽 `block` 按钮紧贴弹窗底边；`DEFAULT_CATEGORY_SEEDS` 已导入却未使用 |

## 二、修复方案（已实施）

### 1. 图表交互（`src/components/charts/BalanceChart.tsx`）
- `roamZoom: false`：停止劫持鼠标滚轮，页面恢复正常滚动。
- 缩放交给**底部 dataZoom 滑块**：拖动两侧手柄缩放窗口、拖动中间平移；`roamDrag: true` 支持在图表内拖拽平移。
- `showDetail` 在手柄上显示当前起止标签，缩放更直观。
- 保留 `onDimensionClick` → 点击任意横轴位置打开当日明细，可改/删（`DayDetailModal`）。
- 更新操作提示文案；保留「重置缩放」（重挂载 `VChart` 复位）。
- 范围选择沿用 `ChartToolbar` 预设（今后 1/2/5/10/20 年、过去~今后、自定义区间）。

### 2. 版面压缩（`src/styles/global.css`、`AccountPanel.tsx`）
- 概览指标条 `.stat`：`padding` 收紧、数值字号 `24px → 20px`。
- 账户面板 `.account-strip`：`flex-wrap: nowrap` + `overflow-x: auto` 单行横向滚动；`.account-card` 改为 `flex: 0 0 auto` 紧凑卡片，余额字号 `18px → 16px`。
- 图表卡片高度 `clamp(440px,64vh,860px) → clamp(360px,52vh,640px)`，让账本明细更早出现。

### 3. 分类预置与弹窗（`CategoryManager.tsx`、`App.tsx`、CSS）
- 复用 `DEFAULT_CATEGORY_SEEDS`，新增「载入常用分类」：按名称去重批量 `addCategory`，无需新增 store 动作。
- 空状态展示「载入常用分类」主按钮 + 引导文案。
- 重构 footer 为 `.category-footer`（左「载入常用分类」、右「新建分类」，`flex` 留出间距），不再贴边。
- 首屏空状态（`App.tsx` onboarding）增加「载入示例数据」按钮，复用 `loadSeed`，`清空` 后可一键恢复示例。

## 五、二期：曲线数据标签与账本明细优化

### 5.1 曲线数据标签（`src/components/charts/BalanceChart.tsx`）
- **可行性**：可行。VChart 折线 `label` 配合 `overlap.hideOnHit` 自动避让：放大后点距变大、标签不再重叠时自动显现。
- **关键实现细节**：使用**条件展开** `...(showLabels && { label: { … } })`，**关闭时整个 label 字段不进入 spec**，避免 VChart 在 `label.visible: false` 路径下产生异常渲染（实测整字段保留 `visible:false` 会让二次开关后曲线短暂消失）。
- **用户开关**：hint 行新增「显示金额 / 隐藏金额」按钮（眼睛图标），**默认关闭**；状态进 `settingsSlice.showChartLabels` 并加入 `persist.partialize` 持久化。
- 样式：`fontSize:10`、`fill: c.axis`；不使用 `stroke` 描边（部分版本下与曲线叠加易出渲染异常），就近紧贴数据点。

### 5.2 账本明细长列表优化（`src/components/charts/LedgerTable.tsx`）
- **问题**：填充数年数据后，按月分组全量渲染（每月一个 Table）导致页面极长、性能差。
- **方案**（两层，互补且简洁）：
  1. 新增「时间范围」筛选 `Select`：近 3 月 / 近 1 年 / 全部，默认**近 1 年**；以「与当前月的月份距离」为窗口（含未来），契合资金推演既看过去也看未来。
  2. 月份分组**可折叠**：折叠的月份只渲染头部（月份 / 笔数 / 净额），不渲染表格 → DOM 与性能可控；默认仅展开「当前月」，其余折叠，点击头部切换。
- 范围内无记录时显示空状态。

### 5.3 二期验证清单

- [ ] 账本默认近 1 年、仅当前月展开；可切换近 3 月/全部；点击月份头部折叠/展开；折叠月份不渲染表格。

## 六、回归修复：账户面板被遮挡

- **现象**：概览条与图表之间的账户行被压成一条窄缝，「现金」卡片完全不可见。
- **根因**：`.app-main` 是 flex column 容器，`.chart-card` 有固定高度，`.account-strip` 的默认 `flex-shrink: 1` 让它在剩余空间不足时被压缩为 0。
- **修复**：给 `.overview-bar`、`.account-strip`、`.ledger-area` 统一加 `flex: 0 0 auto`，固定高度卡片保留固有尺寸不被挤占；`.chart-card` 本身已 `flex: none` 不变。

## 七、Bug 修复：账户编辑初始金额显示为 0 & 图表缩放残留

### 7.1 账户编辑表单初始余额「保存不进 store / 重开显示 0」（`src/components/accounts/AccountFormModal.tsx`）

- **用户反馈现象**：把初始余额改成 10000 → 保存 → 总资产重新计算（用户看到数字变了）→ 再次打开编辑，
  初始余额却又显示 0，数字没记住。
- **真根因**：Semi `InputNumber` 的 `onChange` 在用户键入过程会以 **string 中间态** 触发（例如键入「1」→「10」→「100」→「1000」→「10000」
  期间多次传入 `"1" "10" "100" ..."10000"` 这样的字符串）。
  原代码用 `(v) => setBalance(typeof v === 'number' ? v : 0)` 把所有 string 中间态全部忽略；
  如果用户键入完后**没有先离焦再点保存**（直接键入完→点保存按钮），Semi 不会再补一次 number 回调，
  导致 `balance` state 始终停留在打开时的 0，`handleOk` 把 `yuanToCents(0)=0` 写进 store。
  用户感受到「重新计算了」其实是「旧值被覆盖为 0」造成总资产变化的副作用，并非用户键入的 10000 真的写入了 store。
  Semi 内置 `onNumberChange?: (value: number) => void` 专门用于「只接收有效 number」场景，是这里的正解。
- **修复**：
  - `InputNumber` 由 `onChange` 改为 `onNumberChange={setBalance}`，`balance` 永远是有效 number，
    用户键入流程中的字符串中间态由 Semi 自身处理，组件状态不再丢失；
  - `handleOk` 用 `Number.isFinite(balance) ? balance : 0` 兜底，杜绝 `NaN` / `Infinity` 误写；
  - `useEffect` 依赖从对象引用 `account` 改为值依赖 `[visible, account?.id]`，避免 store 内对象重建触发伪同步；
  - 输入框下方新增 hint：「起始日期当天的账户余额；修改后将重新计算之后所有日期的余额」，明确字段语义。
- **同类反模式连带修复**：项目内 `TransactionFormModal`（金额/间隔/次数 共 3 处）和 `SeriesManageModal`（金额 1 处）
  存在同样的 `onChange + 强转 0/1` 反模式，一并改为 `onNumberChange`，避免相同的「中间态被吞、保存时清零」陷阱。

### 7.2 切换时间范围/粒度后图表保留旧缩放（`src/components/charts/BalanceChart.tsx`）
- **现象**：选「今后 2 年」并把底部 dataZoom 拉到 2 个月窗口；再切到「今后 3 年」，滑块条更新但曲线仍处缩放/裁剪状态，无法看到全部数据。
- **根因**：`dataZoom` 未显式声明 `start/end`，VChart 实例在 spec 变更时保留了上次的缩放百分比；新 values 注入后仍按旧百分比裁剪。
- **修复**：在 `BalanceChart` 内通过 `useEffect([from, to, granularity])` 自动 bump `zoomKey`，
  数据视窗维度变化时强制 VChart 重挂（与「重置缩放」按钮同机制），保证每次切换范围/粒度都从全量视图开始；
  手动缩放仍保留，直到下一次范围/粒度变更或点击「重置缩放」。
  切换可见账户（图例显隐）不重置缩放，保留用户当前观察窗口。

## 八、首屏占比优化：让曲线展示更大

### 8.1 问题
- **现象**：1024×768 视口下，曲线 canvas 仅约 280px（占视口 36%）；概览条、账户行、图表 toolbar 三层加起来占了 ~245px 把主体空间消耗掉，"实际曲线却被极限压缩"。
- **根因**：早期为了卡片视觉松散度，每层都用了大 padding 与大字号；图表卡片用固定 `clamp(360px, 52vh, 640px)`；图表内还有一行教学 hint 文案 + VChart 自身的图例与外部 chips 重复。

### 8.2 修复（`src/styles/global.css`、`AccountCard.tsx`、`BalanceChart.tsx`）
- **概览条 `.stat`**：`padding 10×14 → 8×14`，`value 20px → 18px`，`label 13px → 12px`；高度 ~80 → ~58。
- **账户卡 `.account-card`**：从「头+余额+actions 三段竖排」改为**单行横排**「色点 + 名称 + 余额 + 操作」；高度 ~70 → ~32。
- **账户卡 actions**：去掉「归档」二字标签，三个按钮全部改为 icon-only（编辑 / 归档 / 删除 / 取消归档）+ `Tooltip` 悬停提示，常态 0.55 透明度，hover 1.0；避免文字标签挤压主体。
- **图表卡 `.chart-card`**：由固定 `height: clamp(360px, 52vh, 640px)` 改为 `flex: 1 1 auto; min-height: 480px; max-height: 80vh`，让它**自适应剩余空间**；移动端断点同步放宽到 `min-height: 360px; max-height: 70vh`。
- **图表 toolbar `.chart-toolbar`**：gap `12 → 8`，chips gap `8 → 6`，更紧凑。
- **图表 hint 行**：删除「拖拽底部滑块缩放·…·点击任意位置查看/编辑当日」教学文案，只保留右侧「显示金额 / 重置缩放」两个按钮，整行高度 ~30 → 24。
- **VChart 内置 `legends`**：删除显式配置（图例已由外部 `chart-toolbar__chips` 承担，删除后画布顶部不再有居中的「● 总资产 ● 现金」灰色文字，让出 ~30px 给曲线本身）。
- **`--fm-gap`**：`16 → 12`，全局区块之间间距更紧。

### 8.3 效果
- 1024×768 视口实测：曲线 canvas 从 ~280px 提升到 ~400px（+43%），整体图表卡 ~210→~720 占据主屏中段约 510px；下方账本明细工具栏与首组月份头同屏可见，不再被挤出。
- 移动端断点 (≤768px) 同步收益，且未引入横向滚动。
- 账户名「现金」恢复完整显示；hover 卡片才显操作按钮（半透→不透明），常态视觉更干净。
