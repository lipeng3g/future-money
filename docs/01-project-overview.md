# 项目概述

## 项目名称
**FutureMoney** - 个人财务未来预测工具

## 项目简介
一个基于浏览器的个人财务规划和预测工具，帮助用户可视化未来的现金流和余额走势，提前识别资金紧张或富余时期。

## 核心价值
- 📊 **可视化预测**：通过图表直观展示未来12个月的余额走势
- ⚠️ **提前预警**：识别可能出现资金紧张的时期
- 📈 **趋势分析**：了解收支结构，优化财务决策
- 🔄 **灵活配置**：支持各种周期性和一次性现金流事件

## 目标用户
- 有固定收入和支出的上班族
- 需要规划大额支出的个人
- 希望提高财务意识和规划能力的用户

## 功能范围

### 核心功能
1. **初始资金设置**
   - 设定账户起始余额（如：10,000 元）

2. **现金流事件管理**
   - 一次性事件：某个特定日期的收入/支出
   - 每月事件：每月固定日期的收支（如：10号发工资、20号还房贷）
   - 每年事件：每年固定日期的收支（如：年终奖、汽车保险）

3. **未来余额预测**
   - 自动计算未来12个月每日余额
   - 考虑所有已配置的现金流事件

4. **可视化展示**
   - 余额走势曲线图
   - 现金流堆叠柱状图（月度收支对比）
   - 支持时间轴拖动和缩放

5. **数据持久化**
   - 数据存储在浏览器 localStorage
   - 支持导出为 JSON 文件
   - 支持从 JSON 文件导入

### 暂不实现的功能
- ❌ 标签系统（已简化）
- ❌ 多账户管理
- ❌ 多币种支持（仅支持人民币）
- ❌ 移动端适配（优先桌面端）
- ❌ 后端服务和云同步
- ❌ 用户系统和权限管理

## 技术约束
- **前端框架**：Vue 3 + Composition API
- **状态管理**：Pinia
- **图表库**：ECharts
- **UI组件**：Ant Design Vue
- **日期处理**：date-fns
- **数据存储**：浏览器 localStorage
- **目标浏览器**：现代桌面浏览器（Chrome, Firefox, Safari, Edge）

## 非功能性需求
- **性能**：支持处理至少500个事件，渲染12个月数据
- **可用性**：界面简洁直观，操作流程清晰
- **可靠性**：数据不会因浏览器刷新而丢失
- **可维护性**：代码结构清晰，易于扩展

## 项目里程碑

### 阶段一：基础框架（1-2天）

- 搭建基础布局和路由

### 阶段二：核心功能（3-4天）

- 开发事件管理 CRUD 功能
- 实现余额计算逻辑

### 阶段三：可视化（2-3天）
- 集成 ECharts
- 实现余额曲线图
- 实现现金流堆叠图

### 阶段四：优化与完善（2-3天）
- 数据导入/导出功能
- 性能优化
- UI/UX 细节打磨
- 测试和 Bug 修复
