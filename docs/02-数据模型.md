# 02 · 数据模型

> 核心原则：**周期性变动在创建时即展开为一笔笔真实记录**，绝不在渲染时临时计算。
> 曲线/表格上看到的每一个数据点，背后都有一条可单独修改/删除的 `Transaction`。

## 1. 实体总览

```
Category 1 ──< Account 1 ──< Transaction >── 0..1 Series
                                  │
                          (可选 categoryId)
```

- 一个 `Account` 拥有多个 `Transaction`。
- 一个 `Transaction` 可选属于一个 `Series`（周期组）；一次性变动 `seriesId` 为空。
- `Account` 与 `Transaction` 都可选关联一个 `Category`。

## 2. 类型定义（TypeScript）

```ts
/** 资金变动方向：存入为正，取出为负（内部统一用有符号 amount） */
export type Money = number; // 单位：元，保留 2 位小数，内部以「分」存储避免浮点误差

/** 周期频率 */
export type Frequency =
  | 'once'        // 一次性
  | 'daily'       // 每日
  | 'weekly'      // 每周
  | 'monthly'     // 每月
  | 'quarterly'   // 每季度
  | 'semiannual'  // 每半年
  | 'annual';     // 每年

/** 周期结束条件 */
export type RecurrenceEnd =
  | { kind: 'count'; count: number }        // 重复 N 次
  | { kind: 'until'; date: string };        // 重复到某日期（含）

export interface Category {
  id: string;
  name: string;
  color: string;          // 十六进制色值，用于图表/标签
  createdAt: number;
}

export interface Account {
  id: string;
  name: string;
  categoryId?: string;    // 账户可选归类（如「现金」「理财」）
  openingBalance: Money;  // 初始余额（分）
  openingDate: string;    // 起始日 YYYY-MM-DD，曲线起点
  color: string;          // 图表中该账户线条颜色
  archived: boolean;      // 归档后默认不参与展示
  createdAt: number;
  updatedAt: number;
}

export interface Transaction {
  id: string;
  accountId: string;
  date: string;           // YYYY-MM-DD
  amount: Money;          // 有符号：存入 > 0，取出 < 0（分）
  categoryId?: string;
  note?: string;
  seriesId?: string;      // 属于某周期组则关联，否则为空
  createdAt: number;
  updatedAt: number;
}

/** 周期组：仅保存「生成参数」元数据，便于批量定位与回溯。真实数据在 Transaction 中 */
export interface Series {
  id: string;
  accountId: string;
  frequency: Frequency;
  interval: number;       // 每隔几个周期，默认 1（如每 2 个月）
  baseAmount: Money;      // 生成时的基准金额（有符号）
  startDate: string;      // YYYY-MM-DD
  end: RecurrenceEnd;
  categoryId?: string;
  note?: string;
  createdAt: number;
}
```

### 2.1 持久化根结构

```ts
export interface AppData {
  version: number;            // 数据 schema 版本，便于未来迁移
  accounts: Account[];
  transactions: Transaction[];
  series: Series[];
  categories: Category[];
}

export interface AppSettings {
  theme: 'light' | 'dark';
  rangePreset: string;        // 时间范围预设，如 'P1M-F12M'
  granularity: 'day' | 'week' | 'month';
  visibleAccountIds: string[];// 图表中显示的账户
  showTotal: boolean;         // 是否显示总资产线
}
```

## 3. 字段说明与约束

### Account


| 字段             | 约束                  |
| -------------- | ------------------- |
| name           | 非空，建议唯一（不强制）        |
| openingBalance | 整数（分），可为负           |
| openingDate    | 合法日期；早于此日期不计算余额     |
| color          | 创建时从调色板分配，可改        |
| archived       | 归档账户从图表/默认表格隐藏，数据保留 |


### Transaction


| 字段       | 约束                        |
| -------- | ------------------------- |
| date     | 合法日期；无未来上限（允许记录很远的未来）     |
| amount   | 非 0 整数（分）；正=存入，负=取出       |
| seriesId | 存在则表示由某次周期录入生成；删除该笔不影响其它笔 |


### Series

- `Series` **不是**展示数据来源，仅用于：
  1. 批量管理时定位「同一组」记录；
  2. 展示该组的生成参数（频率/金额/起止）。
- 删除整组 = 删除所有 `seriesId === series.id` 的 `Transaction`（可选连同 `Series` 删除）。
- 单独修改某笔后，该笔仍保留 `seriesId`，**不做特殊「脱离」标记**——因为批量操作采用「按时间列出 + 手动勾选」（见《04》），无需区分是否被改过。

## 4. 金额存储：以「分」为整数存储

为避免 JS 浮点误差（如 `0.1 + 0.2`）：

- 内部所有金额以**整数分**存储与计算；
- 仅在 UI 输入/展示层做「元 ↔ 分」转换；
- 工具函数：`yuanToCents(y)`、`centsToYuan(c)`、`formatMoney(c)`。

## 5. ID 生成

- 使用 `crypto.randomUUID()`；不可用时回退到时间戳 + 随机串。
- 所有实体 id 全局唯一。

## 7. Schema 版本与迁移

- `AppData.version` 当前为 `1`。
- 导入数据时校验 `version`，若低于当前版本，按顺序执行迁移函数 `migrate[v]`。
- 不做「向后兼容的冗余字段」，结构调整通过迁移函数完成（符合前瞻性原则）。
