# 05 · 技术架构与目录结构

## 1. 技术栈


| 领域   | 选型                                            | 说明                    |
| ---- | --------------------------------------------- | --------------------- |
| 语言   | TypeScript 5.x                                | 全量类型                  |
| 框架   | React 18                                      | 函数组件 + Hooks          |
| 构建   | Vite 5                                        | 快速开发、静态产物             |
| UI 库 | `@douyinfe/semi-ui` + `@douyinfe/semi-icons`  | 现代化前台风格、内置深色主题        |
| 图表   | `@visactor/react-vchart`                      | 折线图、多系列、点击交互          |
| 状态   | `zustand` + `persist`                         | 轻量，自动持久化 localStorage |
| 日期   | `dayjs`                                       | 周期步进、格式化              |
| 测试   | `vitest` + `@testing-library/react` + `jsdom` | 算法单测 + 组件测试           |
| 运行环境 | Node 22                                       | 与 `.nvmrc` 一致         |
| 部署   | 静态托管（Cloudflare Pages 等）                      | 纯前端                   |


> 不引入后端；旧版 AI 代理（`functions/api`）本期不保留。

## 3. 状态管理设计（Zustand）

### 3.1 根 store

```ts
type Store = AppData & AppSettings & Actions;

const useStore = create<Store>()(
  persist(
    (set, get) => ({
      ...initialData,
      ...initialSettings,
      ...createAccountActions(set, get),
      ...createTransactionActions(set, get),   // 含 addRecurring / batchDelete / batchUpdate
      ...createCategoryActions(set, get),
      ...createSettingsActions(set, get),
    }),
    { name: 'future-money', version: 1, migrate }
  )
);
```

### 3.2 关键 Actions（接口）

```ts
addAccount / updateAccount / archiveAccount / removeAccount
addTransaction(tx)
addRecurring(input)            // 调 expandRecurrence，写入 series + transactions
updateTransaction(id, patch)
removeTransaction(id)
batchUpdateTransactions(ids, patch)
batchDeleteTransactions(ids)
addCategory / updateCategory / removeCategory
importData(appData) / exportData(): AppData
setTheme / setRange / setGranularity / toggleAccountVisible / toggleTotal
```

### 3.3 派生数据

- 曲线/聚合数据通过 `hooks/useChartData` + `utils` 计算，不存入 store（避免冗余、保持单一数据源）。
- 计算结果用 `useMemo` 缓存，依赖 `transactions / accounts / settings`。

## 4. 持久化与备份

- `persist` 中间件将 `AppData + AppSettings` 写入 `localStorage`（key: `future-money`）。
- 导出：序列化 `AppData` → JSON 文件下载。
- 导入：读取 JSON → 校验 `version`/结构 → `migrate` → 覆盖或合并 → 写回 store。
- 数据量评估：每条变动约 100~~150 字节；上万条约 1~~1.5MB，localStorage（~5MB）可承载。
  - **风险**：若未来需要超大数据量（如多年日频），切换到 IndexedDB（`idb`），store 持久化层抽象为可替换的 `storage` 适配器。

## 5. 主题方案

- Semi 暗色模式通过 `document.body.setAttribute('theme-mode', 'dark')` 切换。
- `useTheme` 读取/写入 `settings.theme`，并同步到 body 属性与 VChart 主题。

## 6. 测试与质量

- `vitest run` 跑算法与 store 单测；核心算法覆盖率优先。
- 关键路径组件测试：表单提交、批量勾选删除。
- ESLint + TypeScript 严格模式；CI 可加 `type-check` + `test`。

## 7. 构建与部署

- `npm run build` 产出 `dist/` 静态资源。
- 部署到任意静态托管（Cloudflare Pages：构建命令 `npm run build`，输出目录 `dist`）。
- 锁定 Node 22（`.nvmrc` / `.node-version`），避免 CI 版本漂移。
