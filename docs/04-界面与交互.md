# 04 · 界面与交互

> 设计基调：**现代化前台（C 端）产品**，不是后台管理系统。
> 以「大图表 + 卡片 + 浮层引导」为核心，避免密集菜单与铺满全屏的表格。

## 1. 设计语言

### 1.1 总体风格

- **图表为视觉中心**：首屏即看到资金走势曲线，第一眼传达「未来我有多少钱」。
- **卡片化**：账户、统计指标用圆角卡片承载，留白充足。
- **浮层引导操作**：新增/编辑/批量管理走 Modal / SideSheet（抽屉），而非整页表单。
- **柔和精致**：圆角（8~16px）、轻微阴影、克制的分隔线、明确的主色与正负色。
- **响应式**：桌面优先，窄屏自适应（卡片堆叠、图表全宽）。

### 1.2 视觉规范（基于 Semi Design Token）


| 项     | 规范                                        |
| ----- | ----------------------------------------- |
| 主色    | 蓝色系（资产/中性正向）                              |
| 收入/存入 | 绿色（`semi-color-success`）                  |
| 支出/取出 | 红色（`semi-color-danger`）                   |
| 圆角    | 卡片 12px，按钮/输入 8px                         |
| 间距    | 8 的倍数（8/16/24）                            |
| 主题    | 深/浅色，使用 `@douyinfe/semi-ui` 主题切换，色板随主题自适应 |
| 字体    | 系统默认 + 数字等宽以对齐金额                          |


### 1.3 明确避免（反例）

- ❌ 左侧多级折叠菜单 + 顶部面包屑的 admin 布局；
- ❌ 一进来就是一张铺满的密集数据表；
- ❌ 每个操作都跳转新页面 / 整页表单。

## 2. 页面布局（桌面）

```
┌────────────────────────────────────────────────────────────┐
│ Header: Logo  | 时间范围 ▾ | 粒度(日/周/月) | 主题 | 导入/导出 │
├──────────────┬─────────────────────────────────────────────┤
│ 账户卡片列   │  概览指标条：当前总资产 / 期末预测 / 月净流   │
│ (可滚动)     │  ┌────────────────────────────────────────┐  │
│ ┌──────────┐ │  │                                        │  │
│ │招行现金   │ │  │        资金走势曲线 (VChart)            │  │
│ │¥20,000 ●  │ │  │  总资产线 + 各账户线，可点击某天        │  │
│ └──────────┘ │  └────────────────────────────────────────┘  │
│ ┌──────────┐ │  图例/显隐开关：●总资产 ●招行 ●理财 …         │
│ │理财       │ ├─────────────────────────────────────────────┤
│ │¥50,000 ●  │ │  账本明细 (按月分组，可筛选)                  │

```

- 窄屏：账户卡片横向滑动条置于图表上方，明细表转为卡片列表。

## 3. 核心交互流程

### 3.1 新建账户

入口：账户区「+ 新建账户」卡片 → 打开 Modal：

- 名称、账户分类（可选）、初始余额、起始日期、配色（调色板）。
- 保存后曲线即从起始日起绘制该账户。

### 3.2 在曲线上点击某天 → 当日明细（DayDetailModal）

- 点击曲线某个数据点（或某账户线某点）→ 弹出该「日期」的明细抽屉/弹窗：
  - 顶部：日期 + 该日各账户余额快照；
  - 列表：当天所有变动（按账户/分类），每条可「改 / 删」；
  - 底部：「+ 新增变动」按钮（预填该日期、该账户）。

### 3.3 新增 / 修改变动（TransactionFormModal）

表单字段：

- 账户（必填）、日期（必填）、**方向开关**（存入/取出）、金额（正数输入，内部转符号）；
- 分类（可选）、备注（可选）；
- **是否周期**（开关）：
  - 关 → 创建 1 笔；
  - 开 → 展开「频率（日/周/月/季/半年/年）+ 间隔 + 结束条件（按次数 / 按截止日期）」，
  实时预览「将生成 N 笔，从 X 到 Y」，确认后展开为真实记录。
- 月末对齐规则在周期区给出说明提示。

### 3.4 批量管理整组（SeriesManageModal）—— 你的勾选方案

入口：任一属于系列的变动行上点「整组」/「管理整组」。

- 弹出该系列**全部记录、按时间升序排列**的列表；
- 每行：复选框 + 日期 + 金额 + 分类 + 备注 + 是否已被单独改过的轻标记（仅提示，不影响操作）；
- **默认全选**；提供「全选 / 反选 / 仅选未来 / 仅选过去」快捷；
- 用户自由勾选后，可执行：
  - **批量删除**（删除选中项）；
  - **批量修改**（对选中项统一改：金额 / 分类 / 备注；日期不批量改）；
- 操作后曲线/表格即时刷新；提供操作结果反馈（Toast）。

### 3.5 表格明细操作

- 行内「改」「删」「整组」三个操作；
- 顶部筛选：账户、分类、时间范围、是否仅看周期；
- 按「年-月」分组小计（当月净流入）。

### 3.6 导入 / 导出

- Header 操作：导出 → 下载 `future-money-YYYYMMDD.json`；
- 导入 → 选择文件 → 校验版本与结构 → 预览摘要（账户数/变动数）→ 确认覆盖或合并。

### 3.7 主题切换

- Header 图标一键切换深/浅；图表色板随主题联动。

## 4. 空状态与引导

- 无账户时：首屏展示友好引导卡片「创建你的第一个账户，开始推演未来」+ 主按钮。
- 有账户无变动：曲线显示为水平的初始余额线，并提示「添加第一笔收支」。

## 5. 反馈与防误操作

- 删除单笔：轻确认（Popconfirm）。
- 批量删除：二次确认，显示「将删除 N 笔」。
- 导入覆盖：明确提示「将覆盖当前数据，建议先导出备份」。
- 周期展开超上限：阻止并提示调整结束条件。

## 6. 组件与界面元素映射（Semi）


| 场景          | Semi 组件                                |
| ----------- | -------------------------------------- |
| 弹窗          | `Modal`                                |
| 抽屉（当日明细/批量） | `SideSheet`                            |
| 表单          | `Form` 系列                              |
| 表格          | `Table`（明细、批量勾选列表）                     |
| 账户卡片        | `Card`                                 |
| 金额/方向       | `InputNumber` + `RadioGroup`/`Switch`  |
| 日期          | `DatePicker`                           |
| 颜色          | 自定义 ColorSwatch（圆点调色板）                 |
| 提示          | `Toast` / `Banner` / `Popconfirm`      |
| 主题          | Semi 暗色模式（`body[theme-mode=dark]`）     |
| 曲线          | `@visactor/react-vchart` 折线图（多系列、点击事件） |
